Built for modern API teams

Every capability you need to go from code to comprehensive, living documentation.

Deep Code Intelligence

DocForge AI's parser understands your entire backend — not just route decorators, but middleware chains, validation schemas, error handlers, and data relationships.

  • Supports 20+ web frameworks automatically
  • Extracts types from TypeScript, Python hints, Java generics
  • Infers auth schemes from middleware patterns
  • Reads existing JSDoc / docstring annotations
  • Detects database models and relationships
// Auto-detected from Express.js
router.post('/users/:id/orders', [
authenticate, validateBody(OrderSchema)
], async (req, res) => {
  // DocForge reads this ↑
  auth: BearerToken,
  params: { id: uuid },
  body: OrderSchema,
  returns: Order | 401 | 422
});

OpenAPI Spec Generation

Generate fully standards-compliant OpenAPI 3.1 specifications with zero manual effort. Every endpoint, schema, and security definition is captured precisely.

  • OpenAPI 3.1 and Swagger 2.0 support
  • Accurate $ref schema components
  • Security scheme definitions (OAuth2, JWT, API Key)
  • Realistic example generation per data type
  • Export as YAML or JSON
paths:
  /users/{id}/orders:
    post:
      security: [{ bearerAuth: [] }]
      requestBody:
        $ref: '#/components/schemas/Order'
      responses:
        201: description: Order created
        422: description: Validation error

Introduction

DocForge AI is an intelligent system that automatically generates comprehensive API documentation, interactive explorers, and client SDKs by analyzing your backend code.

Connect your repository once, and DocForge continuously analyses your route definitions, data models, middleware, and existing annotations to produce living documentation that never goes stale.

Key Capabilities

  • Parse 20+ backend frameworks (Express, FastAPI, Spring Boot, Django, Rails...)
  • Generate OpenAPI 3.1 / Swagger 2.0 / GraphQL SDL specifications
  • Produce client SDKs in TypeScript, Python, Go, Java, Ruby, Rust, Swift, PHP, Kotlin, C#
  • Publish a hosted, searchable documentation portal
  • Auto-update on every Git push via CI/CD hooks

Quick Example

Point DocForge at any backend project and receive full documentation in under 60 seconds:

npx docforge analyze ./src \ --framework express \ --output openapi,sdk:typescript,portal \ --publish

How it Works

DocForge performs multi-pass static analysis: first extracting route definitions, then resolving type information, then inferring auth patterns, and finally synthesizing everything into a coherent OpenAPI specification that drives all other outputs.

// docforge.config.js module.exports = { framework: 'express', entrypoint: './src/routes', output: { openapi: './docs/openapi.yaml', sdks: ['typescript', 'python', 'go'], portal: { publish: true, domain: 'docs.myapi.com' } }, auth: { scheme: 'bearerAuth' } };
